## Данный проект посвящен CI/CD ##

реализован учебный конвейер для утилит cat и grep написанных мной для другого учебного проекта с автоматизацией сборки, проверок и доставки. Конвейер включает:

настройку и регистрацию gitlab-runner на отдельной VM (Ubuntu Server 22.04 LTS) для запуска задач репозитория; регистрация выполняется по URL и токену из задания, runner запускается как служба, чтобы обрабатывать пайплайны, привязанные к проекту.

этап Build: сборка через make (проект SimpleBashUtils), сохранение исполняемых артефактов в выделенную директорию.

этап Style: проверка кодстайла (clang-format) с выводом отчёта прямо в лог задач; при несоответствии — намеренный “fail” пайплайна.

этап Integration tests: запуск интеграционных тестов (собственные для SimpleBashUtils). Этап стартует только если прошли Build и Style, выводит успешность/провал и при ошибках “роняет” пайплайн.

этап Deploy (CD): развертывание на вторую VM (Ubuntu Server 22.04 LTS), запускается вручную и только после успешного CI. Доставка выполняется bash-скриптом по ssh/scp: собранные бинарники копируются на прод-машину в /usr/local/bin. Любая ошибка деплоя помечает задачу как failed.

Уведомления: интеграция с Telegram — бот отправляет статус прохождения и CI, и CD с краткой сводкой (успешно/неуспешно) и ссылкой на репозиторий.


### Part 1. Настройка gitlab-runner

- Поднять ВМ на Ubuntu Server 22.04 LTS
- Скачать и установить на ВМ gitlab-runner 
- Запустить и настроить раннер для использования в текущем проекте

### Part 2. Сборка

- Написать этап CI для сборки приложений из проекта SimpleBashUtils (cat, grep)
	- gitlab-ci.yml должен содержать этап запуска сборки через мейкфайл из проекта 
	- Артефакты должны храниться 30 дней в произвольной директории

### Part 3. Тест кодстайла

- Написать этап CI который будет запускать clang-format
	- Зафейлить пайплайн в случае если кодстайл не прошел
	- Отобразить вывод clang-format

### Part 4. Интеграционные тесты

- Написать этап CI который будет запускать инеграционные тесты
	- Интеграционные тесты уже были написаны в проекте SimpleBashUtils, надо использовать их
	- Этап должен запускаться автоматически только если сборка и clang-format прошли
	- Если тесты не прошли, зафейлить пайплайн
	- В пайплайне отобразить вывод об успешности/неуспешности тестов

### Part 5. Этап деплоя

- Поднять вторую ВМ на Ubuntu Server 22.04 LTS
- Написать этап для CD, который будет разворачивать проект на другой ВМ
	- Запустить этот этап только если все предыдущие прошли успешно
- Написать bash-скрипт который с помощью ssh и scp будет копировать артефакты в директорию /usr/local/bin второй ВМ
- В gitlab-ci.yml добавить этап запуска скрипта
- В случае ошибки надо зафейлить пайплайн

### Part 6. Уведомления

- Настроить уведомления об успешном/неуспешном выполнении пайплайна через бота в Telegram
	- Текст уведомления должен содержать информацию и о этапе CI и о этапе CD
